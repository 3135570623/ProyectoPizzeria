Aplicar el uso de herramientas de control de versiones para gestionar un proyecto de software, utilizando Visual Studio Code y GitHub para crear, modificar
y subir archivos a un repositorio, evidenciando la participación activa de cada integrante mediante commits en el historial del proyecto.


Instrucciones del proyecto
Crear una cuenta o iniciar sesión en GitHub
Cada estudiante debe tener una cuenta activa en GitHub para poder crear o participar en el repositorio del proyecto.

Crear un repositorio en GitHub
Se debe crear un repositorio donde se almacenará el proyecto. Este repositorio servirá para guardar y gestionar todos los archivos del trabajo.

Abrir el proyecto en Visual Studio Code
Utilizar Visual Studio Code como entorno de desarrollo para crear o editar los archivos del proyecto.

Conectar el proyecto con GitHub
Vincular el proyecto local con el repositorio creado en GitHub para poder subir los cambios realizados.

Realizar commits durante el desarrollo
Cada estudiante debe realizar cambios en el proyecto y registrarlos mediante commits.
Se deben realizar al menos tres commits por estudiante para evidenciar la participación.

Subir los cambios al repositorio
Utilizar la opción de push para enviar los cambios realizados desde Visual Studio Code al repositorio de GitHub.

Crear o editar el archivo README
Incluir un archivo README.md donde se explique:

Nombre del proyecto

Objetivo

Descripción del trabajo

Integrantes del grupo

Herramientas utilizadas

Verificar el historial del repositorio
Revisar en GitHub que el historial muestre los commits realizados por cada integrante, como evidencia del trabajo colaborativo.
